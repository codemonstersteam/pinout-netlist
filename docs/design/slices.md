# slices — pinout-netlist

Один внешний вход = один срез. Контракты каждого среза дозаполняются по `program-design` при старте E2 (TBD, один срез = одна ветка = один PR).

| Внешний вход | ID | Срез | Назначение |
|---|---|---|---|
| `POST /reports` | 01 | ingest-report | Принять отчёт валидатора, обновить граф и историю версий |
| `GET /graph` | 02 | query-graph | Отдать текущий граф пар consumer↔provider |
| `POST /impact` (новая версия спеки поставщика) | 03 | diff-breaking-change | Диф контракта против предыдущей версии → список затронутых потребителей |

**Порядок:** 01 → 02 → 03. 01 наполняет хранилище; 02 читает; 03 опирается на накопленную историю версий из 01.

MVP E2 = срезы 01 + 03 (приём отчётов и детект breaking-change). 02 — следом.
