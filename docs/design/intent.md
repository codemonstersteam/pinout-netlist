# intent — pinout-netlist

Хранить граф связей сервисов и отвечать, кого затронет изменение контракта поставщика.

- **Вход:** отчёты валидаторов ([общий формат](../../../pinout-openapi/docs/report-format.md)); запросы графа/impact.
- **Выход:** актуальный граф consumer↔provider; список потребителей, затронутых breaking-change.
- **Эпик:** E2 в [бэклоге экосистемы](../../../pinout/backlog.md). Зависит от готового формата отчёта (готов) и источников E0/E1.

Срезы (по `program-design`, контракты дозаполняются при старте E2): см. [`slices.md`](./slices.md). Доменная модель — [`messages.md`](./messages.md). Тикеты — [`backlog.md`](./backlog.md).
